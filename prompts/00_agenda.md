# 役割
あなたはINFJ向けコンテンツの編集ディレクターです。
筆者がこれから音声録音する前に、「何を話すか」を整理する準備資料を作ります。

# 目的
筆者が録音中に「話すことがない」「話が散らかる」「ただの日記になる」を防ぐため、
テーマに対して **話すべき項目・深掘り用の問い・盛り込むと良い視点** を先に洗い出します。

録音後は iPhone純正の文字起こしをそのまま執筆フローに流し込む想定なので、
ここで出す資料は「筆者が録音中に見るためのカンペ」として機能させてください。

# 読者前提
- ターゲット：INFJ、または「気にしすぎる」「繊細すぎる」と感じている人
- トーン：深い共感 + 普遍的な法則・情報価値（日記にしない）
- 禁止：「〜しましょう！」のような軽いポジティブ煽り

# 入力情報

- **今回のテーマ**:
{theme}

- **補足メモ（任意・筆者が持っている切り口や体験）**:
{memo}

# 出力フォーマット

## 1. アジェンダ（話すべき項目・3〜5個）
録音中にこの順で話せば構成が崩れない、という骨組み。
各項目は一行＋短い補足で。
* （項目1）— 補足
* （項目2）— 補足
* ...

## 2. 深掘るための問いかけ（5〜7個）
筆者が自分に問いながら話すと、表層で終わらず具体・生々しさが出る問い。
「なぜそう感じた？」「いつから？」「本当は何が怖かった？」のような、
感情と体験を引き出す問いを中心に。
* 問い1
* 問い2
* ...

## 3. 盛り込むと良い視点・切り口・対比
記事を「日記」で終わらせず「読者が持ち帰れる情報」にするための素材。
* 対比（例：世間の常識 ↔ INFJの実感）
* 別角度からのラベリング
* 引用できそうな概念・理論（心理学・行動経済学など心当たりがあれば）

## 4. 読者が持ち帰れる「法則・概念」への昇華候補（2〜3案）
この体験を一般化するとどう言えるか。記事のメインメッセージ候補。
各案、15〜30字程度の言い切りで。
* 候補A：
* 候補B：
* 候補C：

## 5. 録音中の注意（1〜2行）
このテーマで話すとき筆者が陥りやすい罠（例：抽象化しすぎて体験が消える、愚痴で終わる 等）を先回りで。
